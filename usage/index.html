Guida aull'uso

Questa sezione delinea le procedure per l'accesso e l'utilizzo delle funzionalità principali del sistema con particolare attenzione ai diversi profili di utenza.

Utente ADMIN

L'accesso al sistema avviene tramite l'interfaccia di login. Per garantire la massima sicurezza e funzionalità, è essenziale comprendere le credenziali di primo accesso e la successiva gestione.

Il sistema di default all'avvio del servizio web, qualora non fosse presente tra gli utenti, creerà un utente predefinito che definisce l'amministratore del servizio, quest'ultimo non farà altro che gestire l'esistenza degli utenti e classi, spetterà ai vari profili docenti dover gestire eventuali presenze, ammonizioni e valutazioni.

NOTA: è vivamente consigliato di modificare le credenziali dell'amministratore creato e proteggerle con dati più sicuri e robusti.

Modifica della password

Modificare la password è fondamentale per una protezione migliore dell'amministratore principale del servizio, per modificarlo basta semplicemente navigare nella directory private/php/database.php e trovare questa sezione, che si trova facilmente tra le prime righe:

if($stmt->num_rows == 0){
    $url_address  = "admin.didattix.it";
    $password     = "admin123";
    $user_type    = "Amministratore";
    $linked_email = "codewithfil@gmail.com";
    $date         = date("Y-m-d H:i:s");

    $username_admin_enc = encrypt($username_admin);
    $password_hashed    = password_hash($password, PASSWORD_DEFAULT);
    $linked_email_enc   = encrypt($linked_email);
    $url_address_enc    = encrypt($url_address);
    $date_enc           = encrypt($date);

    $insert_stmt = $conn->prepare("INSERT INTO users (url_address, nome, password, user_type, linked_email, date) VALUES(?, ?, ?, ?, ?, ?)");
    $insert_stmt->bind_param("ssssss", $url_address_enc, $username_admin_enc, $password_hashed, $user_type,$linked_email_enc, $date_enc);

    if(!$insert_stmt->execute()) echo "Error creating ADMIN user: " . $insert_stmt->error;

    $insert_stmt->close();
}

Modificare quindi $password; effettuate le modifiche eliminare l'utente dalla tabella users, riavviare il servizio web e accedere con nome utente ADMIN e usare la password modificata. L'indirizzo di autenticazione rimane tale. Una volta autenticato con le nuove credenziali potrete usufruire dei privilegi concessi dall'utente amministratore.
